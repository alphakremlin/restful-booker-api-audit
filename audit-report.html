Evidence Collector — Bug Report
Bug #1 — Missing Fields Returns 500 (Server Crash)
Critical
EndpointPOST /booking with {"firstname":"John"} only
Expected400 Bad Request with validation message
Actual500 Internal Server Error — server crashes on bad input
FixAdd input validation middleware. Return 400 with missing field names listed.
Bug #2 — Invalid Date Filter Crashes Server
Critical
EndpointGET /booking?checkin=not-a-date
Expected400 Bad Request
Actual500 Internal Server Error — unvalidated query param crashes endpoint
FixValidate date format before processing. Return 400 with message: "Use YYYY-MM-DD format."
Bug #3 — Failed Login Returns 200 Instead of 401
High
EndpointPOST /auth with wrong credentials
Expected401 Unauthorized
Actual200 OK with {"reason":"Bad credentials"}
FixReturn 401 Unauthorized when credentials are invalid.
Bug #4 — Negative Price Accepted on Booking
High
EndpointPOST /booking with "totalprice": -999
Actual200 OK — booking created with totalprice: -999. No validation on numeric fields.
FixValidate totalprice is a positive number ≥ 0. Return 400 if negative.
Bug #5 — Resource Creation Returns 200 Not 201
Medium
EndpointPOST /booking (successful creation)
DetailReturns 200 OK instead of 201 Created. Violates REST conventions. Breaks automated pipelines that assert correct status codes.
FixReturn 201 Created on successful resource creation.
Bug #6 — X-Powered-By Header Exposes Tech Stack
Medium
DetailX-Powered-By: Express present on all responses. Reveals server technology to attackers. Standard security headers missing.
FixAdd helmet.js middleware to Express. Removes X-Powered-By and sets all security headers automatically.
